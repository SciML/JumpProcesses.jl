135 """
136 view(A, inds...)
137
138 Like [`getindex`](@ref), but returns a lightweight array that lazily references
139 (or is effectively a _view_ into) the parent array `A` at the given index or indices
140 `inds` instead of eagerly extracting elements or constructing a copied subset.
141 Calling [`getindex`](@ref) or [`setindex!`](@ref) on the returned value
142 (often a [`SubArray`](@ref)) computes the indices to access or modify the
143 parent array on the fly. The behavior is undefined if the shape of the parent array is
144 changed after `view` is called because there is no bound check for the parent array; e.g.,
145 it may cause a segmentation fault.
146
147 Some immutable parent arrays (like ranges) may choose to simply
148 recompute a new array in some circumstances instead of returning
149 a `SubArray` if doing so is efficient and provides compatible semantics.
150

153 `SubString`.

159 1 2

164 1
165 3

167 julia> fill!(b, 0)
168 2-element view(::Matrix{Int64}, :, 1) with eltype Int64:
169 0
170 0

172 julia> A # Note A has changed even though we modified b

174 0 2
