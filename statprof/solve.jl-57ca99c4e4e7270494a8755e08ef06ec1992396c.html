762

764 output to give the best interactive user experience, but can be reduced all the

775 between the saved time points.
776 * `saveat`: Denotes specific times to save the solution at, during the solving

783 * `save_idxs`: Denotes the indices for the components of the equation to save.
784 Defaults to saving all indices. For example, if you are solving a 3-dimensional ODE,

816 ### Stepsize Control
817
818 These arguments control the timestepping routines.
819
820 #### Basic Stepsize Control
821
822 These are the standard options for controlling stepping behavior. Error estimates
823 do the comparison
824
825 ```math
826 err_{scaled} = err/(abstol + max(uprev,u)*reltol)
827 ```
828
829 The scaled error is guaranteed to be `<1` for a given local error estimate
830 (note: error estimates are local unless the method specifies otherwise). `abstol`
831 controls the non-scaling error and thus can be thought of as the error around zero.
832 `reltol` scales with the size of the dependent variables and so one can interpret
833 `reltol=1e-3` as roughly being (locally) correct to 3 digits. Note tolerances can
834 be specified element-wise by passing a vector whose size matches `u0`.
835
836 * `adaptive`: Turns on adaptive timestepping for appropriate methods. Default
837 is true.
838 * `abstol`: Absolute tolerance in adaptive timestepping. This is the tolerance
839 on local error estimates, not necessarily the global error (though these quantities
840 are related). Defaults to `1e-6` on deterministic equations (ODEs/DDEs/DAEs) and `1e-2`
841 on stochastic equations (SDEs/RODEs).
842 * `reltol`: Relative tolerance in adaptive timestepping. This is the tolerance
843 on local error estimates, not necessarily the global error (though these quantities
844 are related). Defaults to `1e-3` on deterministic equations (ODEs/DDEs/DAEs) and `1e-2`
845 on stochastic equations (SDEs/RODEs).
846 * `dt`: Sets the initial stepsize. This is also the stepsize for fixed
847 timestep methods. Defaults to an automatic choice if the method is adaptive.
848 * `dtmax`: Maximum dt for adaptive timestepping. Defaults are
849 package-dependent.
850 * `dtmin`: Minimum dt for adaptive timestepping. Defaults are
851 package-dependent.
852 * `force_dtmin`: Declares whether to continue, forcing the minimum `dt` usage.
853 Default is `false`, which has the solver throw a warning and exit early when
854 encountering the minimum `dt`. Setting this true allows the solver to continue,
855 never letting `dt` go below `dtmin` (and ignoring error tolerances in those
856 cases). Note that `true` is not compatible with most interop packages.
857
858 #### Fixed Stepsize Usage
859
860 Note that if a method does not have adaptivity, the following rules apply:
861
862 * If `dt` is set, then the algorithm will step with size `dt` each iteration.
863 * If `tstops` and `dt` are both set, then the algorithm will step with either a
864 size `dt`, or use a smaller step to hit the `tstops` point.
865 * If `tstops` is set without `dt`, then the algorithm will step directly to
866 each value in `tstops`
867 * If neither `dt` nor `tstops` are set, the solver will throw an error.

1409 function check_prob_alg_pairing(prob, alg)
1410 if prob isa ODEProblem && !(alg isa AbstractODEAlgorithm) ||
1411 prob isa SDEProblem && !(alg isa AbstractSDEAlgorithm) ||
1412 prob isa SDDEProblem && !(alg isa AbstractSDEAlgorithm) ||
1413 prob isa DDEProblem && !(alg isa AbstractDDEAlgorithm) ||
1414 prob isa DAEProblem && !(alg isa AbstractDAEAlgorithm) ||
1415 prob isa SteadyStateProblem && !(alg isa AbstractSteadyStateAlgorithm)
1416 throw(ProblemSolverPairingError(prob, alg))
1417 end
1418
1419 if isdefined(prob, :u0) && eltype(prob.u0) <: ForwardDiff.Dual &&
1420 !SciMLBase.isautodifferentiable(alg)
1421 throw(DirectAutodiffError())

1427 throw(NoiseSizeIncompatabilityError(size(prob.noise_rate_prototype, 2),
1428 length(prob.noise.W[1])))
1429 end
1430
1431 # Complex number support comes before arbitrary number support for a more direct
1432 # error message.
1433 if !SciMLBase.allowscomplex(alg)

1435 RecursiveArrayTools.recursive_unitless_eltype(prob.u0) <: Complex

1441 throw(ComplexTspanError())
1442 end
1443
1444 # Check for concrete element type so that the non-concrete case throws a better error
1445 if !SciMLBase.allows_arbitrary_number_types(alg)
1446 if isdefined(prob, :u0)
1447 uType = RecursiveArrayTools.recursive_unitless_eltype(prob.u0)
1448 if Base.isconcretetype(uType) &&
1449 !(uType <: Union{Float32, Float64, ComplexF32, ComplexF64})
1450 throw(GenericNumberTypeError(alg,
1451 isdefined(prob, :u0) ? typeof(prob.u0) :
1452 nothing,
1453 isdefined(prob, :tspan) ? typeof(prob.tspan) :
1454 nothing))
1455 end
1456 end

1459 tType = eltype(prob.tspan)
1460 if Base.isconcretetype(tType) &&
1461 !(tType <: Union{Float32, Float64, ComplexF32, ComplexF64})
